# Решение по прокси для парсинга автосайтов

## ❌ ВАЖНОЕ ИСПРАВЛЕНИЕ

В предыдущем ответе было упомянуто создание **10 поддоменов на Beget для прокси**. Это **НЕВЕРНОЕ** решение для задачи парсинга.

### Почему поддомены на Beget не подходят для парсинга:

1. **Общий IP хостинга** - все поддомены на Beget будут иметь один и тот же IP (или IP из диапазона хостинга)
2. **Не для исходящих соединений** - поддомены нужны для входящих запросов к вашему сайту, а не для исходящих запросов парсера
3. **Блокировка прокси** - китайские/корейские сайты быстро заблокируют IP хостинга Beget

## ✅ ПРАВИЛЬНОЕ РЕШЕНИЕ

Для парсинга che168.com, dongchedi.com, encar.com нужны **внешние прокси-провайдеры**:

### Расчет необходимого трафика

```
Парсинг 3 сайтов:
- che168.com: ~300,000 страниц/день
- dongchedi.com: ~200,000 страниц/день
- encar.com: ~50,000 страниц/день

Итого: 550,000 страниц/день × 30 дней = 16.5M страниц/мес

Средний размер страницы: ~500KB
Трафик: 16.5M × 500KB = 8.25TB/мес

С оптимизацией (кэширование, только изменения):
- Кэширование уже известных страниц: -60%
- Сжатие трафика: -20%
- Итого: ~2.5-3TB/мес

Рекомендуемый тариф: 50-100GB Residential прокси для старта
(парсинг только новых/измененных лотов)
```

## 🔧 Техническая реализация

### Формат подключения прокси

```python
# Пример подключения прокси
PROXY_CONFIG = {
    "provider": "Smartproxy",
    "gateway": "gate.smartproxy.com",
    "port": 10000,
    "username": "sp_user_123456",
    "password": "your_password_here",

    # Ротация IP
    "rotation": "per_request",  # или "per_session"

    # Гео-таргетинг (опционально)
    "country": ["cn", "kr"],  # Китай и Корея
}

# Использование в requests
import requests

proxies = {
    "http": f"http://{PROXY_CONFIG['username']}:{PROXY_CONFIG['password']}@{PROXY_CONFIG['gateway']}:{PROXY_CONFIG['port']}",
    "https": f"http://{PROXY_CONFIG['username']}:{PROXY_CONFIG['password']}@{PROXY_CONFIG['gateway']}:{PROXY_CONFIG['port']}",
}

response = requests.get("https://www.che168.com/", proxies=proxies)
print(response.json())  # или response.text
```

### Проверка IP прокси

```python
import requests

def check_proxy_ip(proxy_url, proxy_port, username, password):
    """Проверка IP прокси"""

    proxies = {
        "http": f"http://{username}:{password}@{proxy_url}:{proxy_port}",
        "https": f"http://{username}:{password}@{proxy_url}:{proxy_port}",
    }

    try:
        response = requests.get(
            "https://api.ipify.org?format=json",
            proxies=proxies,
            timeout=10
        )
        if response.status_code == 200:
            ip = response.json().get("ip")
            print(f"IP прокси: {ip}")
            return ip
    except Exception as e:
        print(f"Ошибка: {e}")
        return None

# Пример использования
check_proxy_ip(
    proxy_url="gate.smartproxy.com",
    proxy_port=10000,
    username="sp_user_123456",
    password="your_password"
)
```

## 📋 План действий

### Шаг 1: Заказ тестовых прокси (1-2 дня)
- [ ] Зарегистрироваться на Smartproxy или IPRoyal
- [ ] Купить тариф 20GB Residential прокси (~$120-150)
- [ ] Получить доступы (username, password, gateway, port)

### Шаг 2: Тестирование прокси (1 день)
- [ ] Проверить работу прокси с Китаем (che168, dongchedi)
- [ ] Проверить работу прокси с Кореей (encar)
- [ ] Замерить скорость и成功率 (success rate)

### Шаг 3: Интеграция в парсер (2-3 дня)
- [ ] Добавить менеджер прокси в код парсера
- [ ] Настроить ротацию IP
- [ ] Добавить обработку ошибок прокси

### Шаг 4: Масштабирование (по мере роста)
- [ ] Увеличить трафик при необходимости
- [ ] Добавить mobile прокси для сложных случаев
- [ ] Настроить гео-таргетинг (китайские IP для che168)

## 📝 Итог

**Поддомены на Beget НЕ НУЖНЫ для парсинга.**

**Нужно:**
1. Купить Residential прокси у специализированного провайдера
2. Интегрировать прокси в код парсера
3. Настроить ротацию IP для обхода блокировок

**Бюджет для старта:** $120-200/мес на прокси
